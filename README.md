#DualCam 
##DualCam — це настільна програма, написана на C++ з використанням Qt 6 та OpenCV. Програма призначена для Raspberry Pi 5 і дозволяє підключати дві камери, порівнювати зображення та відстежувати зміну різкості у реальному часі. Вона використовує GStreamer з libcamera для отримання відеопотоку, Qt для побудови графічного інтерфейсу, а також OpenCV для обробки зображень та побудови графіку.

##Можливості
* Підтримка двох камер. Програма працює з двома модулями камер, підключеними до Raspberry Pi 5. Для кожного кадру обчислюється відносна «різкість» за допомогою оператора Лапласа з гаусівським зважуванням, а результати відображаються на екрані.
* Режими відображення. Доступні два режими:
- Analytics 4Q — у вікні відображаються чотири області: два вихідні кадри, карта різниці між ними та графік історії різкості для обох камер.
- Side by Side — кадри двох камер показуються поруч.
* Побудова графіку історії різкості. Програма накопичує історію відносної різкості кожної камери (до 120 точок) та будує графік у нижній частині вікна. Можна ввімкнути/вимкнути показ графіку.
* Порівняння кадрів. Внизу можна відобразити карту абсолютної різниці між двома зображеннями для швидкого виявлення зміщення чи дефектів.
* Гнучке налаштування. Меню дозволяє вибирати кількість камер (1 або 2), роздільну здатність (640×480 чи 1280×720) та частоту кадрів (30 FPS або 60 FPS), а також вмикати/вимикати показ графіку та карти різниці.
* Гарячі клавіші.
- F — заморозити або розморозити історію (зупинити або відновити накопичення точок на графіку).
- T — показати або сховати графік історії.
- S — переключити режим відображення між Analytics 4Q та Side by Side.
- Esc — закрити програму.
* Меню та повноекранний режим. Меню **Window** дозволяє швидко перейти в повноекранний режим (`Ctrl + F`), повернутись у звичайний (`Esc`), оновити вікно (`Ctrl + R`) або перезапустити додаток.

Ці можливості роблять DualCam зручним інструментом для перевірки фокусу, калібрування камер та візуального порівняння двох потоків на Raspberry Pi 5.

##Встановлення залежностей
На новій системі Raspberry Pi OS виконайте оновлення та встановлення всіх необхідних пакетів. Нижче наведено рекомендаційні команди (потрібні права sudo):
```
bash
sudo apt update
sudo apt full-upgrade    # оновити встановлені пакети

# інструменти для компіляції
sudo apt install build-essential cmake git pkg-config

# Qt 6
sudo apt install qt6-base-dev qt6-tools-dev

# OpenCV для C++
sudo apt install libopencv-dev

# GStreamer: ядро, базові та додаткові плагіни
sudo apt install libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev \
                     libgstreamer-plugins-bad1.0-dev gstreamer1.0-plugins-base \
                     gstreamer1.0-plugins-good gstreamer1.0-plugins-bad \
                     gstreamer1.0-plugins-ugly gstreamer1.0-libav \
                     gstreamer1.0-tools gstreamer1.0-x gstreamer1.0-alsa \
                     gstreamer1.0-gl gstreamer1.0-gtk3 gstreamer1.0-pulseaudio

# libcamera та GStreamer-плагін
sudo apt install libcamera-tools gstreamer1.0-libcamera

# додатково, за потреби Python-байндинги OpenCV
sudo apt install python3-opencv
```
Після встановлення можна протестувати камери:
```
bash
cam -l  # перелік підключених камер

# перевірити роботу плагіна libcamerasrc
gst-launch-1.0 libcamerasrc ! videoconvert ! autovideosink
```
Якщо список камер порожній або виникають помилки, переконайтеся, що модулі камер коректно підключені та активовані у файлі config.txt.

Складання проекту
Код проекту організований за допомогою CMake. Для збірки виконайте такі кроки в терміналі Raspberry Pi:

bash
```
git clone https://github.com/overl00ker/DualCam.git
cd <repository>

# створити каталог збірки та перейти до нього
mkdir build && cd build

# запустити CMake
cmake ..

# зібрати виконуваний файл
make -j$(nproc)

# після успішної компіляції виконуваний файл називається DualCam
```
Якщо CMake не знаходить Qt 6 або OpenCV, переконайтесь, що всі пакети встановлено, а також що відповідні файли cmake доступні у системі. Для Qt 6 зазвичай достатньо встановити `qt6-base-dev`, для OpenCV — `libopencv-dev`.

##Запуск програми
Програма DualCamQt приймає три аргументи командного рядка:
```
bash
./DualCamQt #<ширина> <висота> <частота_кадрів>
```
Приклад запуску з роздільною здатністю 640×480 та частотою 30 FPS:
```
bash
./DualCamQt 640 480 30
```
Після запуску у вікні з’являться потоки від двох камер. Розмір дисплея у режимі Analytics 4Q дорівнює подвоєній ширині та подвоєній висоті вхідних кадрів (тому вказуйте параметри, які підтримують ваші камери). За замовчуванням програма працює з двома камерами; переключити на одну камеру можна через меню Camera → 1 camera. Якщо одна з камер не відкривається, перевірте підключення та опції camera-name.

##Керування з клавіатури та меню
|Комбінація |	Дія |
|-----------|-----|
|F|	Заморожування/розморозка історії (припиняє оновлення графіка)
|T|	Увімкнути або вимкнути показ графіка
|S|	Переключити режим відображення між Analytics 4Q і Side by Side
|Esc|	Закрити програму

Головне меню складається з трьох вкладок:
*Camera — вибір кількості камер, роздільної здатності (640×480 або 1280×720) та частоти кадрів (30 FPS або 60 FPS).
*Advance — ввімкнути/вимкнути показ карти різниці та графіку.
*Window — перейти у повноекранний режим (`Ctrl + F`), повернутися (`Esc`), оновити вікно (`Ctrl + R`) чи перезапустити додаток.

##Типові проблеми
###Камера не визначається
Якщо команда cam -l не показує камеру або GStreamer не знаходить libcamerasrc, переконайтеся, що:
* Ви встановили пакети libcamera-tools та gstreamer1.0-libcamera
* Файл /boot/firmware/config.txt містить відповідні накладки (dtoverlay) для ваших модулів камер, а параметр camera_auto_detect увімкнено (1). Після зміни конфігурації перезавантажте Raspberry Pi.
* Кабелі CSI або USB надёжно з’єднані та не пошкоджені.

###Проблеми з Qt або OpenCV
* CMake не знаходить Qt 6: встановіть `qt6-base-dev` та переконайтесь, що файл `Qt6Config.cmake` знаходиться в `/usr/lib/qt6/` або `/usr/lib/aarch64-linux-gnu/cmake/Qt6*`.
* CMake не знаходить OpenCV: встановіть `libopencv-dev` або зберіть OpenCV із вихідних кодів для актуальної версії. Інсталяція за допомогою `apt-get` спрощує налаштування, але встановлює більш стару версію OpenCV 4.5.1
* Якщо потрібні нові можливості, скористайтеся скриптами для компіляції OpenCV із вихідного коду.

##Високе навантаження на систему
Режим *Analytics 4Q* відображає чотири зображення на високій частоті кадрів, що може навантажувати CPU/GPU Raspberry Pi. Зменшіть роздільну здатність або частоту кадрів через меню *Camera* або запускайте програму із параметрами меншого розміру.

Залежно від версії Raspberry Pi OS та доступних пакетів, назви пакетів можуть дещо відрізнятися. Перевіряйте актуальність через apt search або офіційну документацію.
